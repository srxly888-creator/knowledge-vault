# AI + RPA 深度集成项目

## 项目简介

本项目探索如何让 AI 真正"理解"并"控制" RPA 工具（以影刀为例），实现：

1. **AI 理解业务需求**：解析自然语言，提取关键信息
2. **AI 自动设计 RPA 流程**：将需求转换为可执行的自动化流程
3. **AI 自动调试和优化**：分析执行数据，提供优化建议

## 核心功能

### 1. 自然语言到 RPA 转换器

将用户的自然语言需求自动转换为 RPA 流程代码。

**示例**：
```
输入: "每天早上9点自动登录网站下载报表并发送邮件"
输出: 完整的 RPA 流程代码（包含登录、下载、发送邮件等步骤）
```

### 2. 智能代码生成

- 基于模板的快速生成
- 结合 LLM 的灵活生成
- 自动添加异常处理
- 参数化配置支持

### 3. 性能优化分析

- 自动识别性能瓶颈
- 提供具体的优化建议
- 支持自适应优化
- 实时监控和告警

## 快速开始

### 1. 环境准备

```bash
# 安装 Python 依赖
pip install -r requirements.txt

# 配置环境变量
cp config.example.json config.json
# 编辑 config.json，填入 API 密钥等信息
```

### 2. 基本使用

#### 转换自然语言需求

```bash
python main.py convert \
    -i "每天早上9点自动登录网站下载报表并发送邮件" \
    -o output/my_flow.json
```

#### 执行 RPA 流程

```bash
python main.py execute -f output/my_flow.json
```

#### 优化流程

```bash
python main.py optimize \
    -f output/my_flow.json \
    -l logs/execution.log \
    -o output/my_flow_optimized.json
```

## 核心模块说明

### 1. 自然语言解析模块 (`nl_parser/`)

- **DomainClassifier**：领域分类器，识别需求所属业务领域
- **IntentClassifier**：意图分类器，识别用户想要实现的具体操作
- **EntityExtractor**：实体提取器，提取 URL、时间、路径等关键信息

### 2. 流程规划模块 (`flow_planner/`)

- **StepPlanner**：步骤规划器，将需求分解为可执行的步骤
- **DependencyAnalyzer**：依赖分析器，识别步骤之间的依赖关系

### 3. 代码生成模块 (`code_generator/`)

- **TemplateEngine**：模板引擎，匹配最优的流程模板
- **CodeGenerator**：代码生成器，生成可执行的 RPA 代码
- **StaticCodeChecker**：静态代码检查器，保障代码质量

### 4. RPA 执行模块 (`rpa_executor/`)

- **YingdaoAdapter**：影刀 API 适配器
- **FlowExecutor**：流程执行器
- **TaskScheduler**：任务调度器
- **ExecutionLogger**：执行日志记录器

### 5. 优化分析模块 (`optimizer/`)

- **Profiler**：性能分析器
- **BottleneckDetector**：瓶颈检测器
- **RecommendationEngine**：建议生成引擎

## 实际应用案例

### 案例：电商订单自动化处理

**需求描述**：
> 每天上午10点，从电商平台后台下载前一天的所有订单，进行数据清洗（去除重复、格式化日期、计算总金额），生成销售报表并发送给销售团队。

**AI 自动生成的流程**：

1. 打开电商平台后台
2. 登录系统
3. 导航到订单页面
4. 设置日期筛选（前一天）
5. 下载订单数据
6. 加载 Excel 文件
7. 去除重复订单
8. 格式化日期字段
9. 计算订单总金额
10. 生成销售报表
11. 发送报表给销售团队

**完整实现**请参考技术白皮书第 7 章。
