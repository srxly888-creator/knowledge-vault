## 2. Obsidian AI

### 2.1 平台特色

#### 核心优势
- **本地优先**：所有笔记存储在本地，完全掌控数据
- **插件生态**：1000+ 插件，可高度定制
- **双向链接**：强大的知识网络构建能力
- **Markdown 原生**：纯文本格式，未来兼容性强
- **跨平台**：Windows、macOS、Linux、iOS、Android
- **AI 增强**：通过插件实现强大的 AI 功能

#### 存储架构
```
MyVault/
├── 📄 *.md (笔记文件)
├── 🖼️ attachments/ (附件)
├── 📁 .obsidian/ (配置)
└── 🎨 .obsidian/plugins/ (插件)
```

### 2.2 AI 插件推荐

#### 🔥 Smart Connections（强烈推荐）
- **功能**：AI 驱动的语义搜索和自动链接
- **特点**：
  - 自动检测笔记之间的语义关联
  - 智能推荐相关笔记
  - 基于向量检索的搜索
- **适用**：大型知识库、研究者
- **价格**：免费（本地 LLM）+ 订阅（云端）

#### 🔥 Copilot（强烈推荐）
- **功能**：AI 写作助手
- **特点**：
  - 自动补全、重写、扩写
  - 语法纠错
  - 风格调整
- **适用**：写作密集型工作
- **价格**：免费基础版 + Pro 订阅

#### 🔥 Text Generator AI
- **功能**：多模型 AI 集成
- **特点**：
  - 支持 GPT-4、Claude、Local LLM
  - 自定义 Prompt 模板
  - 批量处理
- **适用**：高级用户、需要多模型
- **价格**：免费 + API 费用

#### 其他优秀插件

| 插件 | 功能 | 适用场景 |
|------|------|---------|
| **QuickAdd** | 快速创建笔记模板 | 日常记录 |
| **Templater** | 高级模板引擎 | 自动化工作流 |
| **Dataview** | 数据查询语言 | 知识库查询 |
| **Excalidraw** | 手绘图表 | 可视化思维 |
| **Canvas** | 可视化知识图谱 | 思维导图 |
| **Advanced Tables** | 表格编辑 | 数据整理 |
| **Mermaid** | 流程图 | 技术文档 |
| **Breadcrumbs** | 面包屑导航 | 知识图谱浏览 |
| **Review** | 间隔重复学习 | 知识巩固 |

### 2.3 最佳实践

#### 工作流设计
```
📥 收集 → 🔗 整理 → 🧠 思考 → 💎 创造 → 🔄 复习
   ↓        ↓        ↓        ↓        ↓
 QuickAdd   双向链接   AI 总结   Copilot   Review
```

#### 笔记结构
```markdown
---
tags: [ai, knowledge-management]
created: 2026-03-25
modified: 2026-03-25
---

# 标题

## 核心概念
...

## 关键要点
- 要点1
- 要点2

## AI 总结
<!-- 使用 Text Generator AI 生成 -->

## 相关链接
- [[相关笔记1]]
- [[相关笔记2]]

## 思考
<!-- 个人见解和思考 -->

---
```

#### AI 工作流
1. **快速记录**：QuickAdd 模板
2. **AI 扩展**：Copilot 扩写和润色
3. **智能链接**：Smart Connections 自动关联
4. **语义搜索**：基于向量检索
5. **间隔复习**：Review 插件巩固记忆

### 2.4 优缺点分析

#### ✅ 优点
- 完全掌控数据
- 插件生态强大
- 高度可定制
- 本地优先，隐私安全
- 跨平台同步（支持 iCloud、Git、Syncthing）

#### ❌ 缺点
- 学习曲线陡峭
- 需要手动配置 AI 插件
- 移动端体验不如原生应用
- 初期配置时间成本高

### 2.5 适用场景

#### ✅ 最适合
- 研究者、学者
- 技术文档编写者
- 知识库建设
- 第二大脑构建
- 需要高度定制的工作流

#### ⚠️ 不适合
- 快速笔记需求
- 移动端重度用户
- 不愿意折腾的用户
- 团队协作（需要额外配置）

## 3. Roam Research

### 3.1 平台特色

#### 核心优势
- **网络化思维**：强大的双向链接和图谱视图
- **Block-level**：最小粒度的编辑单位
- **AI 原生集成**：内置 AI 功能
- **Daily Notes**：每日笔记自动创建
- **查询系统**：强大的数据查询功能

#### 核心概念
```
页面 (Page)
  └─ Block（段落、列表项）
      └─ Block（子块）
          └─ ...
```

### 3.2 AI 功能

#### 🔥 Roam AI（原生）
- **功能**：
  - AI 总结：快速总结长内容
  - AI 问答：基于笔记库回答问题
  - AI 写作：生成内容、扩写、重写
  - AI 搜索：智能语义搜索
  - AI 建议：智能推荐相关内容

- **使用场景**：
  ```
  /ai-summarize [[笔记名]]     # 总结笔记
  /ai-ask 什么是量子纠缠？      # 问答
  /ai-write 扩展这个观点         # 扩写
  /ai-search 相关研究           # 搜索
  ```

#### AI 工作流
1. **Daily Notes**：每天记录想法
2. **AI 总结**：周末用 AI 总结本周内容
3. **AI 问答**：随时向知识库提问
4. **AI 建链接**：AI 自动建议相关链接

### 3.3 核心功能

#### 双向链接
```markdown
# 今天学习了 [[Obsidian]] 和 [[Roam Research]]

它们都有强大的 [[双向链接]] 功能，但是：
- [[Obsidian]] 更加本地化
- [[Roam Research]] 更加云端化
```

#### Block 引用
```markdown
![[今天学习了#^block-id]]  # 引用特定 block
```

#### 查询系统
```clojure
{{query (and (property type "book") (property status "reading"))}}
```

### 3.4 最佳实践

#### Daily Notes 工作流
```markdown
## 📝 今日学习

### 晨间思考
- 记录晨间想法和计划

### 学习笔记
- [[学习主题1]]
  - 要点1
  - 要点2

### 阅读笔记
- [[书名]]
  - 摘录1
  - 摘录2

### 复习
{{query (and (between -7d today) (todo))}}

## 🤖 AI 总结
/ai-summarize 今天
```

#### 知识图谱构建
```
主题 → 子主题 → 具体概念 → 实例
  ↓                      ↓
案例 ← 关联 ← 补充材料 ← 反馈
```

### 3.5 优缺点分析

#### ✅ 优点
- AI 原生集成，开箱即用
- Block-level 编辑灵活
- 知识图谱可视化优秀
- 查询系统强大
- 网络化思维支持好

#### ❌ 缺点
- 云端存储，隐私担忧
- 价格较高（$15/月）
- 移动端体验一般
- 开源生态较弱
- 学习曲线陡峭

### 3.6 适用场景

#### ✅ 最适合
- 创意工作者
- 思想家、作家
- 研究型工作
- 知识图谱构建

#### ⚠️ 不适合
- 预算敏感用户
- 本地数据存储需求
- 团队协作（成本高）

## 4. Logseq

## 5. RemNote

### 5.1 平台特色

#### 核心优势
- **间隔重复**：内置 Spaced Repetition 算法
- **双向链接**：支持知识网络
- **AI 学习助手**：AI 增强的学习体验
- **卡片生成**：一键生成学习卡片
- **多设备同步**：云端同步

#### 学习原理
```
笔记 → 生成卡片 → 间隔重复 → 巩固记忆
```

### 5.2 AI 功能

#### 🔥 AI 卡片生成
- **功能**：
  - 自动从笔记生成学习卡片
  - AI 优化问题设计
  - 智能填空题生成
- **使用**：
  ```
  选择笔记 → AI 生成卡片 → 开始复习
  ```

#### 🔥 AI 学习助手
- **功能**：
  - AI 解释复杂概念
  - AI 生成练习题
  - AI 个性化学习建议
- **适用**：学生、语言学习者

### 5.3 核心功能

#### 卡片类型
```markdown
正面：什么是[[Spaced Repetition]]？
背面：一种基于遗忘曲线的学习算法...

填空：Spaced Repetition 基于__遗忘曲线__
答案：遗忘曲线

关联：[[RemNote]] 使用 [[Spaced Repetition]] 算法
```

#### 学习统计
```
今日复习：50 张
掌握度：85%
遗忘率：5%
```

### 5.4 最佳实践

#### 学习工作流
```
1. 记录笔记
2. 标记重点
3. AI 生成卡片
4. 每日复习
5. 跟踪进度
```

### 5.5 优缺点分析

#### ✅ 优点
- 间隔重复算法科学有效
- AI 学习助手强大
- 适合记忆型学习
- 多设备同步
- 学习统计详细

#### ❌ 缺点
- 云端存储
- 自定义能力有限
- 插件生态弱
- 高级功能需付费

### 5.6 适用场景

#### ✅ 最适合
- 学生
- 语言学习者
- 考试准备
- 技能学习

#### ⚠️ 不适合
- 研究型工作
- 复杂知识管理
- 需要高度定制

## 11. 迁移指南

### 11.1 从 Roam Research 迁移到 Obsidian

#### 步骤
1. **导出 Roam 数据**
   - Settings → Export → Markdown/JSON

2. **转换格式**
   ```bash
   # 使用 roam-to-obsidian 工具
   npm install -g roam-to-obsidian
   roam-to-obsidian input.json output.md
   ```

3. **导入 Obsidian**
   - Create new vault
   - Import converted files

4. **安装插件**
   - Smart Connections（AI）
   - Copilot（AI）
   - Dataview（查询）

5. **调整工作流**
   - Roam block → Obsidian heading
   - Roam query → Dataview query

#### 注意事项
- ⚠️ 双向链接需要重新验证
- ⚠️ Block 引用可能需要手动调整
- ⚠️ 查询语法需要重写

---

### 11.2 从 Notion 迁移到 Obsidian

#### 步骤
1. **导出 Notion**
   - Export → Markdown & CSV

2. **转换格式**
   ```bash
   # 使用 notion-to-md 工具
   npm install -g notion-to-md
   notion-to-md export.zip
   ```

3. **导入 Obsidian**
   - Create new vault
   - Import converted files

4. **安装插件**
   - QuickAdd（模板）
   - Templater（模板）

---

### 11.3 从 Evernote 迁移到 Obsidian

#### 步骤
1. **导出 Evernote**
   - Export → ENEX files

2. **转换格式**
   ```bash
   # 使用 evernote2md 工具
   pip install evernote2md
   evernote2md export.enex
   ```

3. **导入 Obsidian**
   - Create new vault
   - Import converted files
