## 🃏 卡片 3: 提示词工程

### 正面
```
什么是 Prompt Engineering？
核心技巧有哪些？
```

### 背面
```
定义：
通过设计和优化提示词（Prompt），引导大语言模型生成高质量输出的技术与方法。

核心技巧：
1. 清晰具体（明确任务目标和输出格式）
2. 角色扮演（赋予模型特定角色）
3. 示例驱动（Few-shot，提供输入输出示例）
4. 结构化提示（使用分隔符、列表、表格）

高级技巧：
- 思维链 (CoT)：引导模型逐步推理
- 思维树 (ToT)：探索多条推理路径
- 自我反思：让模型评估自己的输出
- 迭代优化：多轮对话逐步完善

常见模式：
- RAG 模式：检索 + 生成
- Agent 模式：规划 + 执行
- 工具调用模式：感知 + 决策 + 执行

相关概念：

## 🃏 卡片 4: RAG

### 正面
```
什么是 RAG（检索增强生成）？
核心技术组件有哪些？
```

### 背面
```
定义：
检索增强生成（Retrieval-Augmented Generation），
结合外部知识库和大语言模型的技术架构。

技术架构：
1. 检索层：从知识库中检索相关信息
2. 增强层：将检索结果注入提示词
3. 生成层：基于增强提示生成回答

关键组件：
- [[向量数据库]]：语义检索
- Embedding 模型：文本向量化
- Chunking 策略：文档分块
- 重排序算法：结果优化

优势：
✅ 知识可更新（无需重新训练）
✅ 减少幻觉（基于事实生成）
✅ 可解释性强（可追溯来源）
✅ 成本低（无需微调）

应用场景：
- 企业知识库
- 智能客服
- 文档问答

相关概念：
- [[向量数据库]]
- [[AI 代理]]
- [[知识图谱]]

来源：YouTube 字幕（30+）
```

## 🃏 卡片 7: 零人 AI 公司

### 正面
```
什么是零人 AI 公司？
核心技术是什么？
与一人公司的区别？
```

### 背面
```
定义：
完全自动化的 AI 驱动公司，
无需人类员工，通过 AI Agent 和自动化工作流实现业务运营。

核心技术：
- Paperclip（开源编排平台）
- AI Agent（自动执行任务）
- 工作流自动化（端到端自动化）
- 函数调用（工具和 API 调用）

与一人公司的区别：
| 维度 | 零人公司 | 一人公司 |
|------|----------|----------|
| 员工数 | 0 人 | 1 人 |
| 自动化 | 100% | 50-80% |
| 编排工具 | Paperclip | 手动管理 |
| 可扩展性 | 高 | 低 |

核心理念：
1. 编排 > 智能（工作流设计比 AI 能力更重要）
2. 自动化 > 人力（尽可能自动化）
3. 系统 > 个人（建立可复制系统）
