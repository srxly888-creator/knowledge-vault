## 核心架构：三层渐进式披露

### 第一层：元数据
- 启动时加载名称和描述
- 装百个Skill不撑爆上下文

### 第二层：指令
- 请求匹配时读取SKILL.md完整内容
- 详细工作流程和规则进入上下文

### 第三层：资源和代码
- 按需读取Markdown、Python脚本、模板
- 进一步节省上下文空间

## 创建Skill

```bash
mkdir -p ~/.claude/skills/my-skill-name
```

SKILL.md结构：
- YAML前置元数据（name, description）
- Markdown正文（具体指令）

## 关键设计模式

### 1. 条件工作流
- 根据输入类型走不同流程
- 分支拆成独立文件

### 2. 反馈循环
- 执行→验证→修复→重试

### 3. 可验证中间输出
- 分析→创建计划→验证→执行→验证

## 实用脚本

预先写好的脚本比临时生成代码更可靠、省token

## 常见坑

1. SKILL.md超过500行（应拆分）
2. 术语不一致
3. 给太多选项
4. 嵌套引用太深

## API集成

- container参数指定技能
- 最多8个技能
- 版本管理重要
- 更改技能列表破坏缓存
